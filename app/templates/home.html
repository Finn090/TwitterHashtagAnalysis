Description of the input fields

Start Date

This field is used to specify a start date of the search. Note that when you choose the 5.10.2020, the newest tweet will be around 4.10.2020 23:59.

End Date

This field is used to specify an end date of the search. Note that if you choose the 5.10.2020, the oldest tweet will be around 6.10.2020 00:00.

Samplesize

Choose this option to limit the query by the number of tweets. If both Enddate and samplesize are chosen, the option which is triggered earlier ends the search.

Seperated Search

Per default, tweets of both queries will be searched simultaneously and combined. The call to the Twitter API will look something like this: query="query1 OR query2". This means that if one hashtag is WAY more popular than the other, you may not get a lot of tweets containing the less popular hashtag. An advantage of this is, that all tweets will be in the same period.
If you choose to make a seperated search both queries will be searched seperately. This means that you will roughly get the same amount of tweets for both hashtags, but for the less popular hashtag the period will probably be longer, because there are less tweets to be fetched per day/hour. So both options have different effects on the outcome of the search.

General Information

The tool takes some time to get all the tweets and do all the computing. The speed depends on the system it is run on. On a fast system (cpu) a query should not take longer than ~5 minutes.
